# Mismo UI Guardrails

## Product Persona

Calm + procedural compliance partner.

Not playful.
Not startup-styled.
Not consumer SaaS.

## Always

- Show timestamps on operational records.
- Keep status labels explicit and procedural.
- Use navy as structural anchor color.
- Use tables for records, audits, and policy references.
- Keep contrast high and legible.
- Use emerald for validated states only.
- Use red for procedural attention states only.
- Keep iconography legal/professional (balance, records, casework).

## Never

- No gradients.
- No oversized rounded corners.
- No celebratory language in risk contexts.
- No floating action buttons for core workflows.
- No playful iconography.
- No aggressive motion.
- No redundant card stacks with duplicate metrics.

## Copy Rules

- Use precise operational language.
- Prefer "requires procedural review" over "needs attention."
- Prefer "Compliance Check-In Required" over "Let's check in."
- Prefer "No active compliance risks detected." over celebratory copy.
- Use "Open Case File" and "View Audit Metrics" as action language.

## Motion Rules

- Transition timing: 120-240ms.
- Hover: slight opacity shift and up to -2px vertical translate.
- No scale bounce.
- No floating/micro-bob animations.

## Dashboard Composition Rules

- Top summary must include a visible composite score.
- Provide score breakdown components and timestamp.
- Timeline modules should be chronological and case-centered.
- Right rail should hold critical health indicators.
- Priority queue must use procedural severity labels.
